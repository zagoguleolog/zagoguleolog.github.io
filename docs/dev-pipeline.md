# Dev Pipeline

## Этапы реализации

1. Анализ задачи и критериев готовности.
2. Проектирование решения и оценка рисков.
3. Реализация минимального рабочего изменения.
4. Тестирование и проверка регрессий.
5. Обновление документации.
6. Подготовка к ревью.

## Артефакты по этапам

- План изменения
- Код и тесты
- Обновлённая документация
- Краткий отчёт о проверках
