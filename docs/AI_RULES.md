# AI Rules

Этот документ — входная точка для AI-агента.

## Порядок работы

1. Прочитать `docs/overview.md`, `docs/domain.md`, `docs/architecture.md`.
2. Прочитать правила из `docs/system/*`.
3. При реализации следовать процессу из `docs/system/pipeline.md`.
4. После изменений выполнить самопроверку и обновить документацию.

## Ссылки на результаты

Если в ответе или в документации упоминается **готовый артефакт** (страница в `app/`, отчёт, собранный файл, опубликованный URL и т.п.), **всегда давай прямую ссылку**, с которой результат открывается одним действием:

- **Локальный файл (предпочтительно для открытия в браузере)** — **абсолютный путь к файлу без префикса `file:`**: пользователь вставляет его в адресную строку Edge/Chrome. Windows: `C:\Users\…\music\app\circle-of-fifths.html` или с прямыми слэшами `C:/Users/…/music/app/circle-of-fifths.html`. Так надёжнее, чем `file:///…`: из чата и из многих приложений клик по `file://` часто блокируется или не срабатывает. Вариант `file:///C:/…` можно добавить вторым, по желанию.
- **Веб** — полный `https://...` без сокращений и без обрезания середины пути.

Относительные пути в репозитории допускаются *дополнительно*, но не вместо прямой ссылки, если пользователю нужно сразу открыть результат.
