# Testing

## Цели

- Подтвердить корректность изменений.
- Предотвратить регрессии.

## Минимальный набор

- Unit-тесты для новой/изменённой логики.
- Integration/API-тесты для межмодульных сценариев.
- Smoke-проверка основных пользовательских потоков.

## Политика

- Любая баг-правка должна содержать тест, воспроизводящий первопричину.
- Если автотестов нет, обязательна ручная проверка с шагами и ожидаемым результатом.
