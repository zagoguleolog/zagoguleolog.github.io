# Development Regulation

## Границы ролей

- Автор изменения отвечает за реализацию и первичную проверку.
- Ревьюер отвечает за качество и риски.
- Решения по архитектуре фиксируются в документации.

## Обязательные правила

- Не маскировать симптомы ошибок; исправлять первопричину.
- Не хранить секреты в коде и документации.
- Поддерживать консистентность кода и документации.

## Post-task flow

1. Самопроверка.
2. Краткий отчёт о проверках.
3. Ревью.
4. Обновление changelog (если применимо).
