# Commenting Standard

## Общие правила

- Комментарии объясняют зачем, а не пересказывают очевидный код.
- Нетривиальные алгоритмы и ограничения должны быть документированы.
- Публичные функции и модули имеют краткое описание контракта.

## Что комментировать обязательно

- Сложные ветвления и алгоритмы.
- Интеграции с внешними API.
- Бизнес-правила и инварианты.
- Побочные эффекты (запись в БД, вызов внешнего сервиса, отправка события).
