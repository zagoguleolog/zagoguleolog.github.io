# Quality Standard

## Архитектура

- Ясные границы модулей.
- Низкая связность, высокая связность внутри модуля.
- Прозрачные интерфейсы.

## Безопасность

- Проверка входных данных.
- Принцип минимальных привилегий.
- Отсутствие секретов в репозитории.

## Поддерживаемость

- Читаемые имена.
- Документированные нетривиальные решения.
- Предсказуемые контракты функций и API.

## Производительность

- Измерение узких мест перед оптимизацией.
- Отсутствие необоснованных тяжёлых операций в горячих путях.
