# Раскладка B-system на баяне (горизонтальное описание)

Термины **класс высоты тона (pitch class, PC)**, **полутон**, **номер ноты MIDI**, научная октава — как в [music-theory.md](music-theory.md) и [domain.md](domain.md).

## Контекст

На схемах учебника клавиатура часто изображена **вертикально** (ряды идут столбцами). Ниже — то же устройство **повёрнутое на бок**: ряды идут **горизонтальными линиями** слева направо, низкий регистр слева, высокий справа (как на правой гриф-клавиатуре при взгляде вдоль корпуса).

Речь о **хроматической** кнопочной системе **B-system** (баян / хроматический аккордеон в раскладке B): **три параллельных ряда** круглых кнопок.

## Теория: шаг вдоль ряда и хроматика по диагонали

1. **Вдоль одного ряда** (горизонтально) соседние активные кнопки отстоят на **малую терцию** (**м3**, 3 **полутона**) — класс высоты тона повторяется через три ступени в октаве (цикл C–D#–F#–A, C#–E–G–A#, D–F–G#–B и т.д.).

2. **Три ряда** сдвинуты так, что вместе покрывают все **12** классов высоты тона:
   - **1-й ряд** (нижний на горизонтальной схеме): стартовый класс **C** (0), далее +**м3** по ряду: … C, D#, F#, A …
   - **2-й ряд** (средний): старт **C#** (1): … C#, E, G, A# …
   - **3-й ряд** (верхний): старт **D** (2): … D, F, G#, B …

3. **Хроматическая гамма** (последовательность соседних **полутонов**) идёт **по диагонали**: от ноты в одном ряду следующий **полутон** — в соседнем ряду со смещением вдоль горизонтали (типичный рисунок «лесенки»).

4. **Нумерация рядов** в русской практике: **1-й**, **2-й**, **3-й** ряд; на горизонтальной схеме сверху вниз удобно подписывать **3-й → 2-й → 1-й** (верхний ряд = 3-й).

## Техническая модель для генерации (скрипт)

Вход: диапазон **номеров ноты MIDI** `midiMin`…`midiMax` (целые, как в [music-theory.md](music-theory.md): **C4 = 60**, **A4 = 69**).

Обозначения:

- `m` — номер ноты MIDI.
- `p = m mod 12` — **PC** (0 = C … 11 = B).
- **Индекс ряда сверху вниз** `r ∈ {0,1,2}` (0 = **3-й** ряд, 2 = **1-й** ряд):

\[
r = 2 - (m \bmod 3)
\]

Проверка: **C** (p = 0) → `m mod 3 = 0` → r = 2 (**1-й** ряд); **C#** → r = 1 (**2-й**); **D** → r = 0 (**3-й**).

- **Хроматическая колонка** (группа для горизонтального смещения «лесенки»):

\[
k = \left\lfloor \frac{m}{3} \right\rfloor
\]

- **Горизонтальная координата** в условных единицах (одна «клетка» — шаг между соседними колонками **k**; доля сдвига внутри колонки задаётся долей от шага):

\[
x_{\mathrm{unit}} = k + (m \bmod 3) \cdot s
\]

где `s` — **доля сдвига по диагонали** (в визуализации по умолчанию `s = 1/3`: три ряда равномерно смещают центры кнопок в пределах одной клетки).

Чтобы поле кнопок начиналось слева от нуля, вычтите минимум `x_unit` по всем кнопкам в диапазоне: `x' = x_unit - x_min`.

**Перевод в пиксели / SVG:**

- `px = padding + x' * cellWidth`
- `py = padding + r * rowPitch` (или с дополнительным вертикальным смещением рядов половиной радиуса, если нужен ступенчатый «кирпичный» вид).

Параметры вёрстки (меняются независимо от музыкальной логики):

- `cellWidth` — горизонтальный шаг колонки;
- `rowPitch` — расстояние между осями рядов;
- `buttonRadius` — радиус круглой кнопки;
- `staggerFraction` — то же, что `s` (0…1, обычно `1/3`).

## Четырёхрядная хроматическая сетка (мод 4)

Это **отдельная** сетка от **B-system**, описанной выше: не переопределяет B-system и не называется им.

**Четыре** параллельных ряда кнопок; **вдоль одного ряда** соседние активные кнопки отстоят на **большую терцию** (**б3**, 4 **полутона**) — шаг по классу высоты тона через четыре ступени в октаве. **Хроматика** (соседние **полутоны**) — **по диагонали** между рядами, по той же идее «лесенки», что у B-system, но с шагом **мод 4** по номеру ноты MIDI.

**Нумерация рядов** (русская практика): **1-й**…**4-й** ряд; на горизонтальной схеме сверху вниз: **4-й → 3-й → 2-й → 1-й**.

Техническая модель (вход тот же — диапазон **номеров ноты MIDI**):

- Пусть `N = 4`. **Индекс ряда сверху вниз** `r ∈ {0,1,2,3}` (0 = **4-й** ряд, 3 = **1-й** ряд):

\[
r = (N - 1) - (m \bmod N) = 3 - (m \bmod 4)
\]

- **Хроматическая колонка:**

\[
k = \left\lfloor \frac{m}{N} \right\rfloor = \left\lfloor \frac{m}{4} \right\rfloor
\]

- **Горизонтальная координата** (доля диагонали `s` в визуализации часто **`1/4`**):

\[
x_{\mathrm{unit}} = k + (m \bmod N) \cdot s
\]

Реализация: тот же модуль [`lib/bayan-b-system.js`](../lib/bayan-b-system.js) с параметром `rowCount` / константой `BAYAN_CHROMATIC_4_ROW_COUNT`; страница [`app/bayan-keyboard.html`](../app/bayan-keyboard.html) — переключатель режима. Физическая клавиатура ПК в режиме **bayiano4** на странице с синтезом — [`app/computer-keyboard-music.mjs`](../app/computer-keyboard-music.mjs): четыре ряда `event.code` сверху вниз начинаются с **`Digit4`**, **`KeyE`**, **`KeyS`**, **`KeyZ`** (см. [synth-structure.md](synth-structure.md)); при охвате октавы 3 от до — стартовые ноты **Eb3 / D3 / C#3 / C3** (4-й…1-й ряд).

## Подписи и цвет (как у классической клавиатуры)

Для отображения имени ноты и октавы в научной нотации: **октава** `n = floor(m / 12) - 1`, **PC** `p = m mod 12`. Буквенная подпись для UI согласована с правилом «белые / чёрные» кнопки:

- **Белая кнопка** (светлый диск): C, D, E, F, G, A, B.
- **Чёрная кнопка**: C#, D#, F#, G#, A# (энгармонически возможны бемоли; для подписи на схеме используются диезные имена).

Реализация: [`lib/bayan-b-system.js`](../lib/bayan-b-system.js). Страница: [`app/bayan-keyboard.html`](../app/bayan-keyboard.html).
